# Narrative Synthesizer：投研叙事融合器

## 定位

本系统处理多篇投研文章、社区文章、产业纪要之间的叙事融合。

它不判断方向真伪，不做外部检索验证，不给可靠性打分。核心问题是：

- 这些文章共同在讲什么方向？
- 哪些主题表面词不同但属于同一个大方向？
- 每篇文章分别补充了什么视角？
- 完整逻辑链如何串起来？
- 哪些产业链环节、公司、催化因素被反复提到？
- 哪些内容只是某篇文章作者自己的延伸？

## MVP 技术栈

- Python
- FastAPI
- LangGraph
- SQLAlchemy
- Pydantic
- DeepSeek / Ollama / Demo LLM Client
- Markdown / JSON 输出

## Workflow

```text
输入文章
  ↓
article_extract_node        单文提取
  ↓
theme_cluster_node          主题聚类
  ↓
theme_merge_node            方向融合
  ↓
angle_compare_node          差异视角比较
  ↓
logic_chain_node            逻辑链重建
  ↓
company_mapping_node        公司/产业链映射
  ↓
report_node                 综合报告生成
```

## 单文提取字段

```json
{
  "article_id": "A",
  "main_themes": ["固态电池", "隔膜材料", "安全性提升"],
  "background": "产业背景",
  "catalysts": ["政策推动", "产业事故", "新技术量产"],
  "industry_segments": ["隔膜", "阻燃材料", "热管理"],
  "companies": ["公司1", "公司2"],
  "logic_chains": [
    "电池安全问题突出 → 隔膜重要性提升 → 高性能隔膜需求增加 → 相关公司受益"
  ],
  "angle": "从安全性和材料升级角度切入",
  "sentiment": "中性",
  "time_window": "中期"
}
```

## 融合主题输出

每个融合主题保留：

- 统一主题名
- 子方向
- 多文共识
- 每篇文章切入角度
- 需要保留的差异视角
- 综合逻辑链
- 上游 / 中游 / 下游产业链映射
- 公司映射
- 反复出现的催化因素

## 报告结构

```text
# 多文章叙事融合报告

## 一、共同关注方向

## 二、方向一：主题名

### 1. 多文共识
### 2. 不同文章的切入角度
### 3. 综合逻辑链
### 4. 涉及产业链环节
### 5. 文章中反复出现的公司
### 6. 需要保留的差异视角
### 7. 反复出现的催化因素
```
